# 就活ES作成支援ツール 要件定義書

## 目的

本ツールは、ユーザーの過去レポート、過去ES、自己PR素材、Codexによる文章生成・改善機能を活用し、応募企業ごとに最適化された就活ESを作成することを目的とする。

単なる文章生成ではなく、企業分析、設問意図の把握、ユーザー経験との接続、文字数調整、表現改善までを一貫して行う。

## ES作成ルール

必ず以下の順番で構成する。

1. 結論
2. 背景
3. 課題
4. 原因分析
5. 行動
6. 結果
7. 学び
8. 企業でどう活かすか

## 文体

- 必ず「です・ます調」
- 受け身は禁止
- 主体的な文章で書く
- 結論を先に書く

## 文字数

- 200字指定: 190〜200字
- 300字指定: 280〜300字
- 400字指定: 380〜400字
- 1000字指定: 900〜1000字

## 数値

成果がある場合は必ず数値を書く。

例:

- 契約数約2倍
- 担当案件3社
- 順位改善
- 九州山口大会3位

## 嘘を書かない

- 実際に経験した内容のみ使用する
- 不足情報は生成しない
- 推測もしない

## 企業別最適化

- コンサル: 論理性、仮説思考、課題解決
- SIer: IT、DX、システム
- デベロッパー: 街づくり、調整力
- エネルギー: 脱炭素、社会課題
- AI企業: AI活用、データ分析、機械学習

## 品質チェック

生成後は必ず確認する。

- 設問に答えている
- 結論が最初
- 企業分析が反映されている
- 人物像が一貫している
- 数値が入っている
- 再現性がある
- 学びがある
- 将来につながる
- 文字数内
- 誤字脱字なし
- 企業名だけ変えた文章になっていない
- 志望理由とガクチカがつながっている
- なぜその行動を選んだかが書かれている
- 関係者との協働が具体的に書かれている
- AI・データ分析経験が活かせる企業では積極的に反映する

## 丸紅サマーインターン ES作成メモ

### ガクチカ 400字

私はドコモショップ営業支援インターンで、顧客課題を起点にした提案力の向上に力を入れました。当初はホームWi-Fiやクレジットカードの商品説明に偏り、契約につながらないことが課題でした。原因は、顧客の通信環境や支払いへの不満を十分に聞き出せず、相手に必要な提案に落とし込めていない点にあると考えました。そこで私は、接客時に利用状況や困りごとを丁寧に伺い、得た情報をもとに店舗の方と提案内容をすり合わせました。また、反応が良かった伝え方を振り返り、次の接客で質問の順番や説明を改善しました。さらに、顧客ごとに重視する点が異なるため、料金、通信速度、手続きの不安などに分けて提案を変えました。その結果、担当当初と比較して契約数を約2倍に伸ばしました。この経験から、成果を出すには相手を深く理解し、関係者と協働しながら本当に必要な価値を形にすることが重要だと学びました。

### 価値観 200字

私が大切にしている考え方は、相手や社会の課題を自分ごととして捉え、立場の異なる人と協働しながら価値を生み出すことです。仕事では、利害や考え方が異なる関係者と向き合い、簡単に答えが出ない課題に取り組む場面が多いと考えます。だからこそ、まず相手を深く理解し、信頼を築き、本質的な課題を見極めて行動したいです。多様な力を結び、社会に必要とされる事業を最後まで形にする姿勢を大切にします。
